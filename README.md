# Inner-SPARC-Portal

## SMTP Email Delivery — Setup & Deployment Guide

The portal sends a **completion email with a PDF certificate attachment** whenever a user completes all steps of the Agent Foundation Training.

### How it works

1. When a user POSTs to `/portal-onboarding/complete/` the view calls `send_onboarding_completion_email_task` (in `app/tasks.py`).
2. A PDF certificate is generated with [WeasyPrint](https://weasyprint.org/) (server-side, no headless browser required) and attached to the email.
3. An inline PNG preview is generated by Playwright when available (optional — falls back gracefully if Playwright is not installed).
4. The email is sent via Django's email backend, configured through environment variables.
5. Duplicate sends are prevented: the `OnboardingProfile.completion_email_status` field tracks `pending / sent / failed / no_email`.

### Required environment variables

Copy `.env.example` to `.env` (local) or set these in your hosting platform's **Secrets / Environment Variables** panel.

#### Gmail SMTP (App Password)

| Variable | Example value | Notes |
|---|---|---|
| `EMAIL_PROVIDER` | `smtp` | Selects SMTP; use `auto` to auto-detect |
| `EMAIL_HOST` | `smtp.gmail.com` | Gmail SMTP server |
| `EMAIL_PORT` | `587` | Use 587 for STARTTLS (recommended) |
| `EMAIL_USE_TLS` | `true` | Must be `true` with port 587 |
| `EMAIL_USE_SSL` | `false` | Set `true` only when using port 465 |
| `EMAIL_HOST_USER` | `you@gmail.com` | Your Gmail address |
| `EMAIL_HOST_PASSWORD` | `xxxx xxxx xxxx xxxx` | 16-char Google App Password |
| `DEFAULT_FROM_EMAIL` | `you@gmail.com` | Displayed "From" address |

#### Google App Password setup

1. Enable **2-Step Verification** on your Google account.
2. Go to **Google Account → Security → App Passwords**.
3. Create an app password for "Mail" / "Other (custom name)".
4. Copy the 16-character code (spaces are optional) into `EMAIL_HOST_PASSWORD`.

> **Why App Passwords?** Google deprecated "Less Secure Apps" access. A dedicated App Password is the correct way to authenticate over SMTP with a Google account.

#### Alternative providers (no custom domain required)

| Provider | Variable | Notes |
|---|---|---|
| Resend | `RESEND_API_KEY=re_…` | Set `EMAIL_PROVIDER=resend` |
| Postmark | `POSTMARK_SERVER_TOKEN=…` | Set `EMAIL_PROVIDER=postmark` |
| Brevo | `BREVO_API_KEY=xkeysib-…` | Set `EMAIL_PROVIDER=brevo` |

### Deployment checklist (why emails vanish after deploy)

- [ ] SMTP environment variables are set in the hosting platform (Render / Heroku / Railway / etc.) — **not just in a local `.env` file**.
- [ ] Outbound SMTP port is not blocked by the host. Use **port 587** (STARTTLS) or **465** (SSL).
- [ ] The Google App Password is still valid (passwords can be revoked when 2FA settings change).
- [ ] `EMAIL_PROVIDER` is set, or at minimum `EMAIL_HOST_USER` **and** `EMAIL_HOST_PASSWORD` are non-empty (triggers `auto` selection of SMTP).
- [ ] `DEFAULT_FROM_EMAIL` matches the authenticated SMTP sender.

### Local development

```bash
# Install dependencies
pip install -r requirements.txt

# Copy and fill in environment variables
cp .env.example .env
# Edit .env with your credentials

# Run migrations
python manage.py migrate

# Start the development server
python manage.py runserver
```

To verify email sending without a real SMTP server, set:

```env
EMAIL_BACKEND=django.core.mail.backends.console.EmailBackend
```

Emails will be printed to the console instead of being delivered.

### Running tests

```bash
python manage.py test app.tests
```

The test suite includes `TestOnboardingCompletionEmail` which verifies:
- Completion email is sent to the user's account.
- A PDF certificate (`certificate.pdf`) is attached.
- Missing email address results in `no_email` status (no crash).
- Provider failures are handled gracefully with `failed` status.
- Non-delivery backends (console) correctly report failure.
